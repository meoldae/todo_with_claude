# CLAUDE.md

이 파일은 Claude Code(claude.ai/code)가 이 저장소에서 작업할 때 참고하는 안내 문서입니다.

## 프로젝트 개요

**투두 관리**와 **가계부**를 통합하고 연말정산 기능을 지원하는 개인 생산성 앱. 전체 요구사항은 `requirements.md` 참고.

## 개발 로드맵

| 단계 | 범위 | 플랫폼 |
|------|------|--------|
| 1단계 | 투두 앱 | PC 앱 (Windows) |
| 2단계 | 가계부 + 연말정산 | PC 앱 (Windows) |
| 3단계 | 모바일 확장 | Android / iOS |

## 핵심 도메인 개념

- **Todo**: 제목, 메모, 마감일, 우선순위, 카테고리, 반복 일정, 완료 상태를 가진다.
- **가계부**: 금액, 날짜, 카테고리, 결제 수단(신용카드 vs 체크카드)을 포함한 수입·지출 내역을 기록한다.
- **연말정산**: 의료비·교육비·기부금·카드 사용액 등 특정 지출 카테고리가 공제 항목에 매핑되며, 연간 한도 대비 누적액을 추적하고 보고서를 내보낸다.

## 로컬 가이드 참조 규칙

- 백엔드 작업 시 → `backend/claude-local.md` 를 반드시 읽고 시작한다.
- 프론트엔드 작업 시 → `frontend/claude-local.md` 를 반드시 읽고 시작한다.

## 아키텍처 제약

- **1·2단계는 서버 없이 로컬 기기에 데이터를 저장한다.** 3단계 모바일 확장 시 BaaS(Firebase, Supabase 등) 도입을 검토한다.
- 로컬 저장 방식이더라도 엔티티 구조와 API 규격은 처음부터 공유 백엔드를 전제로 설계한다. 나중에 서버로 전환할 때 클라이언트 코드 변경을 최소화하기 위함이다.
- 3단계까지 PC(Windows)·Android·iOS가 동일한 기능을 제공해야 하므로, 플랫폼별로 다른 데이터 모델을 만들지 않는다.
- 연말정산 소득공제율 계산을 위해 모든 지출 항목에 신용카드/체크카드 구분이 필수다.
