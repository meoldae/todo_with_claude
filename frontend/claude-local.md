# Frontend — Claude Local Guide

프론트엔드 개발 시 참고하는 안내 문서. 프로젝트 전체 맥락은 상위 디렉토리의 `CLAUDE.md`와 `requirements.md` 참고.

## 프론트엔드 역할

- 사용자가 투두·가계부 데이터를 생성·조회·수정·삭제하는 UI 제공
- 백엔드 API와 통신하여 데이터 동기화
- 플랫폼별 UX에 맞게 화면을 구성 (PC Windows 앱 → 추후 Android/iOS 확장)

## 플랫폼 목표

| 단계 | 플랫폼 | 비고 |
|------|--------|------|
| 1·2단계 | Windows 데스크톱 앱 | 마우스·키보드 중심 UX |
| 3단계 | Android / iOS | 터치 중심 UX, 푸시 알림, 위젯, OCR 영수증 입력 |

## 주요 화면 구성

### 투두
- **Today 뷰**: 오늘 마감 항목 + 우선순위 높은 항목 요약
- **전체 목록 뷰**: 카테고리·우선순위·날짜별 필터, 키워드 검색, 드래그 앤 드롭 순서 변경
  - 각 항목 맨 앞에 체크박스 배치 — 클릭/터치 한 번으로 완료 토글 (Microsoft Todo 방식)
  - 완료 항목은 취소선·흐림 처리 후 목록 하단에 분리 표시
- **상세/편집**: 제목·메모·마감일·우선순위·반복 설정
- **통계**: 주간·월간 완료율

### 가계부
- **월별 내역 뷰**: 수입·지출 목록, 월간 수지 요약
- **예산 현황**: 카테고리별 예산 대비 지출 진행률 시각화
- **통계**: 카테고리별·월별 지출 추이 그래프, 전월 대비 증감
- **연말정산 대시보드**: 공제 항목별 누적액 / 한도 현황, 신용카드·체크카드 비율

## UI 원칙

- 투두와 가계부는 하나의 앱 안에서 탭 또는 사이드바로 전환
- 연말정산 대시보드는 가계부 섹션 내 별도 뷰로 제공
- 다크 모드를 기본 지원 (라이트/다크 토글)
- 3단계 모바일 확장을 고려해 레이아웃 컴포넌트는 플랫폼 분기가 가능한 구조로 설계

## 데이터 흐름

- 모든 데이터는 백엔드 API를 통해 관리 (로컬 단독 저장 없음)
- 오프라인 상태에서의 임시 저장 및 재연결 시 동기화 처리 고려
- 결제 수단(신용카드/체크카드) 구분은 거래 입력 시 필수 선택 항목으로 처리
