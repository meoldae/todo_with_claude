# Backend — Claude Local Guide

백엔드 개발 시 참고하는 안내 문서. 프로젝트 전체 맥락은 상위 디렉토리의 `CLAUDE.md`와 `requirements.md` 참고.

## 현재 저장 방식

- **1·2단계: 로컬 저장** — 별도 서버 없이 기기 로컬에 데이터를 저장한다.
- **3단계 이후**: BaaS(Firebase, Supabase 등) 도입하여 모바일 동기화 지원 예정.
- 전환 비용을 줄이기 위해 지금부터 엔티티 구조와 API 규격은 서버 연동을 전제로 설계한다.

## 백엔드 역할

- 투두·가계부 데이터의 저장, 조회, 수정, 삭제 처리
- (3단계 이후) PC(Windows) 앱과 모바일(Android/iOS) 간 데이터 동기화 제공
- 연말정산 공제 항목 집계 및 보고서 생성
- (3단계 이후) 계정 인증 및 사용자별 데이터 격리

## 핵심 도메인 모델

### Todo
| 필드 | 설명 |
|------|------|
| id | 고유 식별자 |
| title | 제목 |
| memo | 메모 |
| due_date | 마감일 |
| priority | 우선순위 (높음/보통/낮음) |
| category_id | 카테고리 참조 |
| repeat | 반복 설정 (없음/매일/매주/매월) |
| is_completed | 완료 여부 |
| user_id | 소유 사용자 |

### 가계부 (Transaction)
| 필드 | 설명 |
|------|------|
| id | 고유 식별자 |
| type | 수입 / 지출 |
| amount | 금액 |
| date | 거래일 |
| category_id | 카테고리 참조 |
| payment_method | 신용카드 / 체크카드 / 현금 / 기타 |
| tax_deduction_type | 연말정산 공제 항목 (해당 없음/의료비/교육비/기부금 등) |
| memo | 메모 |
| user_id | 소유 사용자 |

### 연말정산 공제 항목 (TaxDeductionSummary)
- 연도별, 공제 유형별 누적 금액 집계
- 공제 한도 초과 여부 판단은 백엔드에서 처리

## API 설계 원칙

- 모든 엔드포인트는 인증된 사용자만 접근 가능
- PC·모바일 클라이언트가 동일한 API를 공유 (플랫폼별 별도 API 없음)
- 동기화 충돌 발생 시 최신 수정 시각(updated_at) 기준으로 처리

## 연말정산 관련 비즈니스 규칙

- 신용카드 소득공제율: 15%, 체크카드·현금: 30% (기준은 변경될 수 있으므로 하드코딩 금지, 설정값으로 관리)
- 의료비 공제 가능 기준: 총급여의 3% 초과분
- 공제 한도는 항목별로 상이하며, 연도별 세법 개정에 따라 달라질 수 있으므로 별도 테이블로 관리
