# 요구사항 검토 및 추가 제안

## 놓친 부분

### 투두 — UX 공백

- **빠른 입력**: 항목 추가 시 매번 모달/폼을 열지 않고, 목록 하단 입력창에 텍스트 치고 Enter로 바로 추가하는 흐름 없음.
- **하위 할 일(서브태스크)**: 큰 할 일을 작은 단계로 쪼개는 기능 없음. Microsoft Todo의 핵심 기능 중 하나.

### 가계부 — 실사용 공백

- **고정 지출 자동 등록**: 월세, 구독료 같은 고정 지출을 매달 자동으로 기록하는 기능 없음. 고정 지출 등록은 있지만 자동 반복 입력이 명시되지 않음.
- **잔액 개념 부재**: 수입·지출만 기록하고 "이번 달 남은 돈"이나 전체 잔액 표시 없음.

### 연말정산 — 실질적 공백

- **총급여 입력 기능**: 의료비 공제 기준이 "총급여의 3% 초과분"인데 사용자가 총급여를 입력하는 화면이 요구사항에 없음. 없으면 계산 자체가 불가능.
- **부양가족 입력**: 인적공제, 의료비·교육비 공제는 부양가족 수에 따라 한도가 달라지는데 이 개념이 빠져 있음.

## 추가하면 좋을 기능

### 투두

- **중요 표시(별표/즐겨찾기)**: Microsoft Todo의 별표 기능. 우선순위와 별개로 "지금 당장 눈에 띄어야 하는 것" 표시용으로 유용.
- **마감일 없는 항목 구분 뷰**: 날짜 있는 것과 없는 것을 섞으면 관리가 어려워짐.

### 가계부

- **자주 쓰는 거래 즐겨찾기**: 반복적으로 입력하는 거래(편의점, 카페 등)를 템플릿처럼 저장해 빠르게 입력.
- **월별 목표 저축액 설정**: 예산 관리에 지출 한도만 있고 "이번 달 얼마 모을 거야"라는 목표 개념 없음.

### 통합 (투두 + 가계부 연계)

- **할 일에 지출 연결**: "병원 예약" 투두를 완료하면서 의료비를 바로 가계부에 입력하는 흐름. 두 기능이 통합 앱인 만큼 이 연결이 차별점이 될 수 있음.
